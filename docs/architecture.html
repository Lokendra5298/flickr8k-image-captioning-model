Model Architecture

The model has two main parts: a CNN encoder that understands the image, and an LSTM decoder that writes the caption.

The CNN extracts visual features. The LSTM uses those features and previous words to predict the next word.

Part 1: CNN Encoder

The encoder answers one question: what is inside the image? Instead of generating words directly, it converts the image into numbers that summarize the important visual information.

What happens inside the encoder?

  1. The input image is resized and normalized.
    The model expects a tensor shaped like [batch_size, 3, 224, 224], where 3 means RGB channels.
  2. ResNet18 extracts visual patterns.
    Early convolution layers detect simple edges and colors. Deeper layers detect object parts, scenes, and shapes.
  3. The final classification layer is removed.
    ResNet18 normally predicts ImageNet classes. Here, we do not need class names. We only need visual features.
  4. The ResNet weights are frozen.
    Frozen means the pretrained CNN is used as a feature extractor and its parameters are not updated during training. This makes training faster and safer on a small dataset.
  5. A linear layer maps features to the caption embedding size.
    The 512-D ResNet output is passed through a fully connected layer, ReLU, and dropout before going to the decoder.
Simple meaning: the encoder turns a picture into a compact visual memory that the decoder can use while writing the caption.

Part 2: Word Embedding Layer

Neural networks cannot directly understand words like man, dog, or snow. So every word is converted into a number ID, and then that ID is converted into a dense vector using an embedding layer.

For example, the caption may be represented like this during training: 

<start> a man is standing in the snow <end>

During training, the decoder sees previous words and learns to predict the next word.

How the image enters the LSTM

In this implementation, the image feature vector is used to initialize the LSTM hidden state and cell state. This is important because the LSTM starts sentence generation with memory of the image. 

image_features --> init_h --> initial hidden state
image_features --> init_c --> initial cell state

What the LSTM learns

  • Common caption patterns, such as a man is or a dog is running.
  • Which words usually follow other words.
  • How to condition sentence generation on the visual feature vector.
  • When to stop by predicting the <end> token.

Part 4: Linear layer and softmax

The LSTM output is not a word yet. It is a hidden vector. A final linear layer converts this hidden vector into one score for every word in the vocabulary. 

LSTM hidden vector --> Linear(hidden_size, vocab_size) --> word scores

During training, cross-entropy loss compares these scores with the correct next word. During prediction, the model chooses a word from these scores, usually the highest-scoring word in greedy decoding.

Training mode vs prediction mode

Mode What the model receives What the model does
Training Image plus the correct caption tokens Learns to predict the next correct word at each position.
Prediction Image plus only the <start> token Generates one word, feeds it back, then repeats until <end> or max length.
Important: a model can have good token accuracy but still generate imperfect captions. Captioning quality is usually better judged with examples and caption metrics such as BLEU, METEOR, ROUGE, CIDEr, or SPICE.
