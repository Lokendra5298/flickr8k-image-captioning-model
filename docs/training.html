Training Guide

This page explains the full training pipeline in easy words: dataset, preprocessing, vocabulary, model training, checkpoint saving, and metric plotting.

Step 1: Install the project

git clone https://github.com/Lokendra5298/flickr8k-image-captioning-model.git
cd flickr8k-image-captioning-model
pip install -r requirements.txt

Recommended packages include Python, PyTorch, torchvision, numpy, pandas, and matplotlib.

Step 2: Download the Flickr8k dataset

Run the dataset download script:

python src/download_dataset.py

After download, your dataset should look like this:

data/flickr8k/flickr8k/
├── Images/
└── captions.txt

Verify the files:

find data/flickr8k -type f -name "*.jpg" | head
find data/flickr8k -type f -name "captions.txt"

Step 3: Dataset path configuration

By default, the code searches inside data/flickr8k. If your dataset is somewhere else, set the environment variable: 

export FLICKR8K_DIR=/path/to/your/flickr8k
Example: if your dataset folder contains flickr8k/Images and flickr8k/captions.txt, then set FLICKR8K_DIR to the parent folder that contains that flickr8k directory.

Step 4: What preprocessing does

Training an image captioning model needs both image preprocessing and text preprocessing.

Image preprocessing

  • Load image from Images/.
  • Resize or crop to 224 x 224.
  • Convert to tensor shape 3 x 224 x 224.
  • Normalize so the pretrained ResNet18 receives expected input values.

Text preprocessing

  • Read captions from captions.txt.
  • Lowercase and tokenize words.
  • Build a vocabulary: word to ID and ID to word.
  • Add special tokens such as <start>, <end>, <pad>, and possibly <unk>.

Step 5: How one training example works

Suppose the image shows a person standing in snow and the target caption is: 

<start> a man is standing in the snow <end>

The model receives the image and the caption tokens. It learns to predict the next word at every position:

Input so far Target next word
<start> a
<start> a man
<start> a man is
<start> a man is standing in the snow <end>

This is why training is a next-word prediction problem guided by image features.

Important training concepts

Cross-entropy loss

This loss compares the predicted vocabulary distribution with the correct next word. Lower loss means better next-word prediction.

Token accuracy

This measures how often the model predicts the correct next word token. It is useful, but it is not the same as full caption quality.

Train/validation/test split

Training data teaches the model. Validation data helps monitor generalization. Test data gives the final held-out performance estimate.

Overfitting

If training accuracy keeps improving but validation/test accuracy stops improving, the model is memorizing training captions more than generalizing.

Step 7: Checkpoints and logs

After training, the model and logs are saved here:

checkpoints/caption_model.pth
logs/training_log.csv

The checkpoint stores learned model weights. The CSV log stores epoch-by-epoch training, validation, and test loss and accuracy.

Step 8: Plot loss and accuracy curves

After training, generate plots with:

python src/plot_metrics.py

The plots used in this blog are placed at:

assets/images/accuracy_curve.png

In GitHub Pages, the files are loaded from the blog folder using assets/images/loss_curve.png and assets/images/accuracy_curve.png.

Step 9: Recommended experiments

Experiment Why it helps
Unfreeze later ResNet layers Lets the image encoder adapt to Flickr8k images instead of using frozen ImageNet features only.
Use beam search instead of greedy decoding Explores multiple caption candidates and often improves sentence quality.
Add BLEU and CIDEr evaluation Token accuracy is limited. Caption metrics compare generated sentences with reference captions.
Use attention Allows the decoder to focus on different image regions for different words.
Try a stronger encoder ResNet50, EfficientNet, ViT, or CLIP features can produce richer visual representations.
