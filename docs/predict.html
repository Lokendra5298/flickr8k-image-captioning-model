Generate a Caption for a New Image

After training, the model can take a single image and generate a caption word by word.

Basic prediction command

Use the prediction script with an image path:

python src/predict.py --image data/flickr8k/flickr8k/Images/1356796100_b265479721.jpg

If your script supports automatic image search, you may also pass only the image filename.

python src/predict.py --image 1356796100_b265479721.jpg

What happens during prediction?

  1. Load the image.
    The image is resized, normalized, and converted to a tensor.
  2. Extract image features.
    ResNet18 converts the image into a feature vector.
  3. Initialize the LSTM.
    The image feature vector is used to initialize the LSTM hidden state and cell state.
  4. Start with <start>.
    The decoder receives the start token and predicts the first word.
  5. Repeat word by word.
    The predicted word is fed back into the LSTM to predict the next word.
  6. Stop at <end>.
    Generation stops when the decoder predicts the end token or reaches the maximum length.

Greedy decoding explained

The current decoder uses greedy decoding. This means the model chooses the highest-probability word at every step. 

current word: <start>
model predicts probabilities for all words
choose highest probability word: a

current word: a
choose highest probability word: man

current word: man
choose highest probability word: is

repeat until <end>
Limitation: greedy decoding is simple and fast, but it can miss better full sentences. A word that is best now may not lead to the best complete caption.

Example output

For a snowy image, the model may generate:

a man is standing in the snow

This sentence is produced one token at a time. The model does not copy it from a database; it predicts each word based on the image feature vector and the words generated so far.

How to improve prediction quality

Method Simple explanation Expected result
Beam search Keep several possible captions at each step instead of only one. Usually more fluent captions.
Temperature sampling Control how random the word choice is. Can produce more diverse captions.
Attention Let the decoder look at different image regions for different words. Better object and location descriptions.
Better encoder Use ResNet50, EfficientNet, ViT, or CLIP features. Stronger image understanding.

Checklist before predicting

  • The dataset vocabulary or saved vocabulary file is available.
  • The checkpoint exists at checkpoints/caption_model.pth.
  • The input image path is correct.
  • The same preprocessing used during training is used during prediction.
  • The model is in evaluation mode using model.eval().
